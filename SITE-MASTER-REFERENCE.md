## 5. Checkout Flow

### Overview
The checkout follows a **custom on-site intermediate step** strategy (Maximus-style), capturing only contact info before handing off to Shopify's native checkout for shipping and payment.

### Flow Diagram

```
User clicks "Start My Protocol" (or any product CTA)
  ‚Üí addItem() called on Zustand store
  ‚Üí Shopify Storefront API cartCreate/cartLinesAdd mutation
  ‚Üí Cart drawer opens automatically (isCartOpen = true)
  ‚Üí User clicks "Proceed to Checkout"
  ‚Üí Navigates to /checkout (on-site page)
  ‚Üí User enters email + optional phone
  ‚Üí updateCartBuyerIdentity() called (email/phone only)
  ‚Üí window.open(checkoutUrl, '_blank') ‚Üí Shopify native checkout
  ‚Üí Shopify handles shipping address, payment, order confirmation
  ‚Üí useCartSync hook detects completed order on tab return ‚Üí clears cart
```

### Key Implementation Files

| File | Purpose |
|---|---|
| `src/lib/shopify.ts` | Storefront API client, GraphQL mutations, cart CRUD functions, buyer identity update |
| `src/stores/cartStore.ts` | Zustand persistent store ‚Äî items, cartId, checkoutUrl, all cart operations |
| `src/hooks/useCartSync.ts` | Syncs cart on page load and `visibilitychange` events |

- Cart is created on first `addItem` call via `cartCreate` mutation
- Subsequent items added via `cartLinesAdd`
- Cart auto-clears on "cart not found" errors (expired carts)
- `syncCart()` runs on page load and tab visibility change ‚Äî clears cart if Shopify reports `totalQuantity === 0`
